BOM 数据模型设计

产品 BOM = 树形节点（制造层级）+ 行项目（物料清单）

核心概念

BOM 节点树（sp_product_bom）
🏭 台式电脑主机 [level=0, 产品]
  └─ 🔧 台式电脑半成品 [level=1, 半成品]
      ├─ 📦 主板单元 [level=2, 组件]
      └─ 📦 机箱单元 [level=2, 组件]

每个节点代表制造过程中的一个步骤
节点具有父子关系，构成完整的制造层级

BOM 行项目（sp_product_bom_item）
主板单元 的物料清单：
  ├─ CPU i7-13700K    x1
  ├─ DDR5 32GB        x2
  ├─ SSD 1TB NVMe     x1
  ├─ 主板 Z790         x1
  └─ 散热器            x1

机箱单元 的物料清单：
  ├─ 机箱外壳 ATX      x1
  ├─ 电源 750W 金牌    x1
  └─ 散热风扇 120mm    x3

台式电脑半成品 的物料清单：
  ├─ 主板单元           x1 ← 引用BOM节点
  └─ 机箱单元           x1 ← 引用BOM节点

每个节点有对应的物料清单
行项目可引用物料或其他BOM节点

表结构设计

sp_product_bom（BOM 节点表）
字段 类型 说明
idvarcharSnowflake 主键
bom_codevarcharBOM编码，自动生成 PBOM-XXX
product_codevarchar产品物料编码（根节点）
node_namevarchar节点名称（产品名/半成品名/组件名）
parent_idvarchar父节点ID（null=根节点）
levelint层级 0=产品 1=半成品 2=组件
versionvarchar版本号，如 V1.0
statusvarchardraft=草稿 / locked=已锁定
remarkvarchar备注信息
locked_atdatetime锁定时间
locked_byvarchar锁定人
sp_product_bom_item（BOM 行项目表）
字段 类型 说明
idvarcharSnowflake 主键
bom_idvarchar所属 BOM 节点 ID
item_typevarcharmaterial=物料 / bom_ref=BOM节点引用
material_codevarchar物料编码
material_descvarchar物料描述
quantitydecimal用量
unitvarchar单位

需要确认的问题

关于 BOM 行项目的物料来源：

A

从物料主数据中选择

BOM 行项目的物料从已有的"物料信息定义"（sp_materile）中选取，确保数据一致性

B

可以手动输入 + 选择物料

既可以关联已有物料，也可以手动输入临时物料名称（灵活性更高）
